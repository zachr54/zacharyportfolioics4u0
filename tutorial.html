Tutorial: Object-Oriented Programming

An overview of the core concepts I learned in the Object-Oriented Programming unit.

What is OOP?

Object-Oriented Programming (OOP) is a way of writing code by grouping related data and methods together into classes. Instead of having separate variables and functions scattered everywhere, OOP organizes a program around objects. This makes code easier to read, reuse, and update—especially in larger programs like the bank account system we built in class.

Here is a simple example from class. The Test class stores a string and splits it into words. The printWords method loops through and prints each one. In main, we create an object and call that method on it.

Test.java 
public class Test {
    private String str = "hello_my_name_is_zach";
    private String[] words = str.split("_");

    public void printWords() {
        for (String word : words) {
            System.out.print(word + " ");
        }
    }

    public static void main(String[] args) {
        Test t = new Test();
        t.printWords();  // Output: hello my name is zach
    }
}

Common Mistakes & How to Avoid Them

  • God Classes: Creating one class that tries to handle everything in the program.
    How to avoid: Give each class one clear purpose instead of putting all the logic in one place.
  • Ignoring Encapsulation: Making all variables public so any part of the program can change them directly.
    How to avoid: Keep fields private and use getters and setters to control access. This protects the data inside your objects.
  • Over-using Inheritance: Building long chains of parent and child classes that become hard to follow.
    How to avoid: Use inheritance when it makes sense, like when SavingsAccount extends BankAccount because a savings account is a type of bank account.

A World Without OOP

Without OOP, programs would rely more on long lists of separate functions and global variables. In big projects, this can get messy fast—functions start changing data they were not meant to touch, and it becomes harder to find and fix bugs. OOP helps keep related code together through encapsulation, allowing it to interfere as little as possible with other aspects of the program.
